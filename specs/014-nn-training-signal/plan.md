# Implementation Plan: NN Training Signal Improvement

**Branch**: `014-nn-training-signal` | **Date**: 2026-03-14 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/014-nn-training-signal/spec.md`

## Summary

Refactor-first approach to improving NN training signal for aircraft path-following. Phase 1 strips GP dependencies and simplifies autoc into a standalone NN-only system. Phase 2 adds training improvements (sigma floor, curriculum, fitness decomposition, sep-CMA-ES). The refactoring enables fast iteration on training approaches by removing modal branching, GP infrastructure leakage, and unnecessary complexity.

## Technical Context

**Language/Version**: C++17 (g++, CMake 3.10+)
**Primary Dependencies**: Eigen3 (linear algebra), AWS SDK (S3), inih (config parser, vendored). Boost removed during refactoring.
**Storage**: S3 for evolution archives, local files for diagnostics (data.dat, data.stc)
**Testing**: GoogleTest 1.14.0, ctest
**Target Platform**: Linux (Ubuntu 22.04), Arduino/Xiao BLE Sense (xiao-gp export)
**Project Type**: CLI (evolution engine + simulation workers + tools)
**Performance Goals**: 50 population x 49 scenarios = 2,450 sims per generation; wall-clock target: < current 5000-pop GA
**Constraints**: 531-dimensional weight space, NN01 serialization format preserved, sensor-in/control-out contract preserved
**Scale/Scope**: 3 repos (GP/autoc, CRRCSim, xiao-gp), ~15k LOC in autoc

## Implementation Phases

### Phase A: Dependency Analysis & Contract Tests
1. Full dependency analysis: `nm libgp.a`, transitive include tracing, symbol scanning
2. Contract tests for: evaluator (sensor-in/control-out), config (key parsing), evolution loop (gen-over-gen), RPC transport (send/receive)
3. These tests define surviving behavior вЂ” refactoring must not break them

### Phase B: Strip GP, Replace Config, Simplify Transport
1. Remove GP tree eval (autoc-eval.cc), bytecode interpreter (gp_bytecode.cc/h), gpextractor from autoc
2. Break inheritance: MyGP no longer inherits GP, MyPopulation no longer inherits GPPopulation, MyGene no longer inherits GPGene вЂ” replace with standalone classes or thin wrappers around NNPopulation
3. Replace GPConfiguration/GPVariables with inih (vendor into third_party/) вЂ” affects config_manager.h/cc, autoc.cc, nnextractor.cc
4. Replace GPrand()/GPsrand() with std::mt19937 вЂ” affects autoc.cc, pathgen.cc, gp_math_utils.h
5. Remove all `#include "gp.h"` and `#include "gpconfig.h"` from 11 files (see research.md table)
6. Remove libgp.a link, `include_directories(../include)`, `link_directories(../lib)` from CMakeLists.txt
7. Remove GP deserialization from minisim.cc, renderer.cc (these only need NN archive reading)
8. Strip modal if(GP)/if(NN) branches вЂ” happy path only
9. Verify: autoc, minisim, renderer all build and run with `ctest`

### Phase C: Replace Boost (trivial components)
1. boost::thread в†’ std::thread
2. boost::mutex/cv в†’ std::mutex/cv
3. boost::format в†’ sprintf/std::format
4. boost::date_time в†’ std::chrono
5. boost::process в†’ fork()/execv()

### Phase D: Replace Boost (medium components)
1. boost::asio в†’ POSIX sockets wrapper (~150 LOC)
2. boost::log в†’ simple custom logger
3. boost::iostreams в†’ std::stringstream

### Phase E: Replace Boost (serialization) + Simplify Transport
1. Custom binary RPC format (see contracts/rpc-transport.md)
2. Remove Boost serialization from EvalData, EvalResults, AircraftState
3. Single unified transport for train and eval
4. NN01 format preserved (already Boost-free)

### Phase F: Source Reorganization
1. Create include/autoc/{nn,eval,util}/ directory structure
2. `git mv` all headers and source files
3. Update CMakeLists.txt include paths
4. Update all #include directives
5. Move test binaries to build/tests/

### Phase G: Cross-Repo Updates
1. CRRCSim: update minisim RPC to match new transport format
2. xiao-gp: verify nnextractor в†’ nn2cpp в†’ build pipeline
3. Remove #ifdef GP_BUILD guards from shared evaluator code

### Phase H: Run Output Management
1. Configurable output directory (auto-created per run)
2. Remove hardcoded eval- prefix
3. All run artifacts (data.dat, data.stc, logs, weights) in run subdir

### Phase I: Repo Extraction
1. Extract autoc/ to standalone repository
2. Verify standalone build (no parent GP references)
3. Migrate relevant specs/docs

### Phase J: Training Signal Improvements
1. Sigma floor (FR-001) вЂ” clamp mutation sigma to configured minimum
2. Curriculum scenario ramp (FR-002) вЂ” wire VariationRampStep to NN evolution
3. Per-scenario fitness decomposition (FR-003) вЂ” minimax/percentile aggregation
4. sep-CMA-ES optimizer (FR-004) вЂ” ~200-300 LOC custom implementation
5. Per-timestep fitness streaming (FR-007) вЂ” extend RPC response
6. Per-segment credit assignment (FR-005) вЂ” segment scoring on timestep data
7. Checkpoint/resume (FR-008) вЂ” save full evolution state per generation
8. Behavioral cloning bootstrap (FR-006) вЂ” supervised MSE from GP flight data
